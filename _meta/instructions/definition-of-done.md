# Definition of Done

**Type:** knowledge
**Summary:** What "done" means for each content type — prevents draft accumulation by defining clear completion criteria.
**Tags:** #meta #process
**Status:** active
**Updated:** 2026-04-07

---

## The Problem

It's easy to create drafts. It's hard to ship them.

When you work with AI to create documents, guides, and decisions, they often stay as drafts — never published, never shared, never actually "done."

This defines **what "done" actually means**, so you can distinguish between:
- "We drafted something" (not done)
- "It's live and useful" (actually done)

---

## The Rule: Nothing is "Done" Until It's Useful

A document is NOT complete until someone (including your future self) can find it and use it.

- Status = Draft → Still working on it
- Status = Active → Actually done

---

## Definition of Done by Content Type

### Knowledge Articles (How-tos, Guides, References)

**Done when:**
- [ ] Content is complete and accurate
- [ ] Metadata block is filled in (Type, Summary, Tags, Status, Updated)
- [ ] Filed in the correct directory (not still in `_inbox/`)
- [ ] Status changed from Draft to Active
- [ ] Related links added if connections exist

### Decision Records

**Done when:**
- [ ] Decision clearly stated with rationale
- [ ] Options considered and trade-offs documented
- [ ] Owner and date recorded
- [ ] Status set to Active or Complete
- [ ] Filed in `2-Knowledge/Decisions/`

### Initiatives / Projects

**Done when:**
- [ ] All goals/success criteria met (or explicitly descoped)
- [ ] Artifacts shipped to their final locations
- [ ] Status changed to Complete
- [ ] Initiative folder moved to `1-Projects/_SHIPPED/`
- [ ] Wiki log updated

### Meeting Notes

**Done when:**
- [ ] Key discussion points captured
- [ ] Action items listed with owners
- [ ] Filed (not in inbox)

---

## How to Use This

### During a Session
Your AI should nudge you toward completion:
- "This is ready — should we change status to Active?"
- "Want to file this from the inbox to its final location?"
- "All checklist items are done — marking as Complete."

### The Completion Moment
When ALL checklist items are done:
1. Status updated to Active/Complete
2. Filed in the correct directory
3. Wiki log updated if it's a significant addition

---

## Red Flags

These phrases mean something is NOT done:
- "Guide drafted" → Not done until filed and active
- "Decision recorded" → Not done until rationale is documented
- "Notes captured" → Not done until action items are assigned

---

## The Point

**The goal isn't to have more drafts. The goal is to build useful, findable knowledge.**

A guide in `_inbox/` helps nobody. A guide in `2-Knowledge/HowTo/` with proper metadata helps everyone — including your future self.

Draft ≠ Done. Active = Done.
