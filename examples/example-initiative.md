# Onboarding Documentation Overhaul

**Type:** initiative
**Summary:** Rewriting all onboarding docs from scratch - current docs are outdated and new hires consistently report confusion in their first week.
**Tags:** #initiative #onboarding #documentation
**Status:** active
**Owner:** Engineering Manager
**Updated:** 2026-03-28
**Related:** [example-knowledge-article.md](example-knowledge-article.md)

---

## Problem Statement

New hires take 2-3 weeks to become productive. Exit surveys from the last 3 hires all mention "confusing onboarding docs" as a pain point. Current docs reference deprecated tools, missing steps, and assume knowledge that new hires don't have.

---

## Goals & Success Criteria

- [ ] New hire can set up dev environment in < 2 hours (currently ~6 hours)
- [ ] All onboarding docs pass accuracy review (no references to deprecated tools)
- [ ] Next hire rates onboarding docs 4+ out of 5

---

## Scope

**In scope:**
- Dev environment setup guide
- Architecture overview for new engineers
- Team processes and conventions
- Tool access and permissions guide

**Out of scope:**
- HR onboarding (handled by People team)
- Product training (handled by Product team)
- Customer-facing documentation
