## Overview

The TMC5160A is a combined stepper motor controller and gate-driver IC targeting applications from 8 V to 60 V where coil currents exceed what an integrated H-bridge can handle. All power switching is done by eight external N-channel MOSFETs; the IC provides gate drive, current sensing, protection logic, and a hardware-only motion controller. The A-version is a pin- and software-compatible drop-in replacement for the TMC5160, fixing an automatic-tuning detection limitation in the non-A version. It is also register-compatible with the [[TMC2209]]-family lower-current devices and the [[TMC4671-LA]] closed-loop controller.

Three operation modes are selected via pins SD_MODE and SPI_MODE:

| Mode | SD_MODE | SPI_MODE | Description |
|------|---------|----------|-------------|
| Full motion controller | 0 | — | Internal ramp generator drives all motion; SPI or UART for config/status |
| Step/Dir + SPI/UART | 1 | 1 | External step pulses; SPI or UART still accessible for tuning |
| Standalone Step/Dir | 1 | 0 | Fully pin-configured; no SPI/UART required |

## Architecture

**Controller block (ramp generator):** A dedicated hardware arithmetic unit generates trapezoidal or six-section velocity profiles, manages reference switch events, encoder deviation checking, and automatic current scaling. No firmware runs on the IC — all real-time functions are pure hardware.

**Driver block:** Gate-drive circuits for eight external N-channel MOSFETs (two full H-bridges). An internal 11.5 V gate voltage regulator (12VOUT) supplies the low-side drivers; bootstrapped capacitors on CA1/CB1/CA2/CB2 supply the high-side drivers. At VS ≥ 48 V an external gate supply to the 12VOUT pin is recommended to reduce internal dissipation. Configurable break-before-make (BBM) timing prevents shoot-through. Gate drive strength is programmable (DRVSTRENGTH in DRV_CONF 0x0A): 200 mA / 400 mA / 600 mA (low-side, DRVSTRENGTH 0/2/3).

## SixPoint Ramp Generator

The ramp generator (Ch. 12 of the datasheet) executes moves in hardware in under one microsecond after a register write. Velocity is accumulated every 512 clock cycles. All velocity values use internal units; conversions use fCLK as the reference:

- **Velocity:** v[Hz] = v[5160] × (fCLK / 2 / 2²³)  
- **Acceleration:** a[Hz/s] = a[5160] × fCLK² / (512 × 256 × 2²⁴)  
- **Time base:** t = 2²⁴ / fCLK

### Ramp Registers

| Register | Address | Width | Description |
|----------|---------|-------|-------------|
| RAMPMODE | 0x20 | 2-bit | 0=positioning, 1=velocity+, 2=velocity−, 3=hold |
| XACTUAL | 0x21 | 32-bit signed | Actual position |
| VACTUAL | 0x22 | 24-bit signed | Actual velocity (read-only) |
| VSTART | 0x23 | 18-bit | Start velocity (unsigned) |
| A1 | 0x24 | 16-bit | First acceleration (VSTART→V1) |
| V1 | 0x25 | 20-bit | Threshold velocity; 0 disables two-phase ramp |
| AMAX | 0x26 | 16-bit | Second acceleration (V1→VMAX); sole accel in velocity mode |
| VMAX | 0x27 | 23-bit | Target/max velocity; range 0–(2²³−512) |
| DMAX | 0x28 | 16-bit | Deceleration VMAX→V1 |
| D1 | 0x2A | 16-bit | Deceleration V1→VSTOP; must not be 0 in positioning mode |
| VSTOP | 0x2B | 18-bit | Stop velocity; must be ≥ VSTART; min 10 recommended; 100–1000 for fast termination |
| TZEROWAIT | 0x2C | 16-bit | Wait time at zero velocity before reversing |
| XTARGET | 0x2D | 32-bit signed | Target position (write to start move) |

**RAMP_STAT (0x35)** carries position_reached, velocity_reached, vzero, event_stop_l/r, event_stop_sg, second_move, t_zerowait_active, and status_latch_l/r flags.

**VDCMIN (0x33):** Setting ≥ 256 enables dcStep above this velocity threshold; 0 disables dcStep in controller mode.

## Interfaces

### SPI
40-bit datagrams (1 address byte + 4 data bytes), MODE 3, MSB first. Every datagram returns an 8-bit status byte: `reset_flag | driver_error | sg2 | standstill | velocity_reached | position_reached | status_stop_l | status_stop_r`. Multiple devices can be daisy-chained on a single bus.

### UART (single-wire)
CRC8-ATM protected, auto baud rate detection (9600 to fCLK/16). Up to 255 slaves addressable via NAI/NAO chaining pins; slave address set in SLAVEADDR (0x03). Read accesses require a separate reply frame.

### Step/Dir
REFL_STEP and REFR_DIR pins accept standard step-and-direction signals. `dedge` bit in CHOPCONF (0x6C) enables double-edge step counting, doubling effective step rate. MicroPlyer interpolation (CHOPCONF bit 28 `intpol`) scales any input resolution up to 256 µsteps/step. Minimum STEP pulse width: max(tFILTSD, tCLK+20) ns ≈ 100 ns typ. DIR setup/hold to STEP: 20 ns.

### ABN Encoder
Pins ENCA, ENCB, ENCN (not available in standalone Step/Dir mode). 32-bit counter X_ENC (0x39); ENC_CONST (0x3A) is a signed 16.16 fixed-point prescaler mapping encoder pulses to motor microsteps. ENC_DEVIATION (0x3D) triggers an interrupt when motor-encoder mismatch exceeds the threshold. Maximum counting rate < 2/3 fCLK. ENCMODE (0x38) configures polarity, N-channel behavior, and latching options.

### Reference Switches
REFL_STEP (left) and REFR_DIR (right) double as reference switch inputs. SW_MODE (0x34) configures stop_l/r_enable, polarity, swap_lr, latch_l/r_active/inactive, sg_stop (StallGuard2 auto-stop), en_softstop, and en_latch_encoder. XLATCH (0x36) captures XACTUAL at the switch event for sub-step-accurate homing.

**Homing procedure (switch-based):**
1. Move away from switch; arm latch and soft-stop in SW_MODE.
2. Ramp toward switch — switch hit → position latched in XLATCH, motor decelerated.
3. Wait for standstill (vzero flag or VACTUAL = 0).
4. Switch RAMPMODE to hold (3); compute offset between XLATCH and XACTUAL; write correction to XACTUAL.

**Homing with StallGuard2:** Use hard stop (en_softstop = 0); enable sg_stop in SW_MODE; set TCOOLTHRS; clear event_stop_sg in RAMP_STAT to restart after stall.

## Motion Features

### StealthChop2
Voltage-mode PWM current control — silent, resonance-free operation. Enabled by GCONF bit 2 (`en_pwm_mode`). Key PWMCONF (0x70) parameters: PWM_OFS [7:0] (default 30), PWM_GRAD [15:8], PWM_FREQ [17:16] (chopper freq), pwm_autoscale (bit 18), pwm_autograd (bit 19), freewheel [21:20], PWM_REG [27:24] (regulation speed, default 4), PWM_LIM [31:28] (amplitude limit on mode switch, default 12). Read back auto-tuned values from PWM_SCALE (0x71) and PWM_AUTO (0x72). Active when TSTEP ≥ TPWMTHRS (0x13).

**Automatic tuning (AT):** AT#1 — motor at standstill at IRUN for ≤ 130 ms sets PWM_OFS_AUTO; AT#2 — motor moving at 60–300 RPM sets PWM_GRAD_AUTO. IRUN ≥ 8 required. TPOWERDOWN (0x11) minimum 2 required; allow 100 ms standstill before first move in StealthChop.

### SpreadCycle
Hysteresis current-mode chopper (CHOPCONF chm = 0). Parameters: TOFF [3:0] (1–15; 0 = driver off), TBL [16:15] (blank time 16/24/36/54 clocks), HSTRT [6:4] (hysteresis start), HEND [10:7] (hysteresis end). Active when TSTEP < TPWMTHRS or when StealthChop is disabled.

### StallGuard2
Load-angle measurement updated every fullstep. SG_RESULT [9:0] in DRV_STATUS (0x6F) reads 0 at maximum load / stall, up to 1023 at no load. Tuning: adjust sgt [22:16] in COOLCONF (0x6D) — range −64 to +63; start at 0; decrease for higher sensitivity, increase for less. Enable filtered mode (sfilt bit 24) for CoolStep; disable for fast stall response. Active between TCOOLTHRS (0x14) and THIGH (0x15). Engage sg_stop in SW_MODE to halt on stall. Best results at 30–70 % of nominal current, 1–5 RPS. SG measurement error ≤ ±max(1, |SGT|).

### CoolStep
Load-adaptive current reduction. Enabled by setting SEMIN [3:0] > 0 in COOLCONF (0x6D). When SG_RESULT < SEMIN×32 → current increases; when SG_RESULT > (SEMIN+SEMAX+1)×32 → current decreases. SEUP [6:5] sets current increment step (1/2/4/8); SEDN [14:13] sets number of above-threshold readings per decrement step (32/8/2/1). SEIMIN (bit 15): minimum current 1/2 or 1/4 of IRUN. Reduces power consumption up to 75 %. Tune SGT before enabling CoolStep.

### dcStep
Load-adaptive automatic commutation mode — the motor never loses steps. Enabled by VDCMIN (0x33) ≥ 256 in controller mode, or DCEN pin in Step/Dir mode. Requires vhighfs and vhighchm bits set in CHOPCONF; TOFF > 2 recommended. DC_TIME (0x6E [9:0]) sets commutation reference pulse width (lower limit: TBL blank time + 1–100 clocks); DC_SG (0x6E [23:16]) controls stall sensitivity (set ~DC_TIME/16). LOST_STEPS (0x73) counts skipped steps for later compensation in Step/Dir mode.

### Passive Braking and Freewheeling
Controlled via PWMCONF freewheel [21:20]: 0 = normal, 1 = freewheel, 2 = low-side short (passive braking), 3 = high-side short. Also configurable through IHOLD = 0 combined with FREEWHEEL mode.

### Resonance Dampening
TPFD [23:20] in CHOPCONF activates passive fast decay pulses (0 = off; 1–15 × 128 clocks ≈ 10 µs per step) to suppress mid-range resonances.

## External MOSFET Requirements

Eight N-channel MOSFETs in two full H-bridges. Standard 10 V gate-threshold MOSFETs recommended; logic-level MOSFETs are compatible but more prone to cross-conduction. Select DRVSTRENGTH (DRV_CONF 0x0A [19:18]) based on MOSFET Miller charge:

| Miller charge | Recommended DRVSTRENGTH |
|--------------|------------------------|
| < 10 nC | 0 |
| 10–20 nC | 0 or 1 |
| 20–40 nC | 1 or 2 |
| 40–60 nC | 2 or 3 |
| > 60 nC | 3 |

BBM timing configured by BBMTIME [4:0] (100–375 ns) and BBMCLKS [11:8] in DRV_CONF. SHORT_CONF (0x09): S2VS_LEVEL [3:0] sets low-side short sensitivity (4 = highest … 15 = lowest); S2G_LEVEL [11:8] sets high-side sensitivity (min 12 if VS can exceed 52 V). SHORTFILTER [17:16]: 0 = 100 ns, 1 = 1 µs (default), 2 = 2 µs, 3 = 3 µs.

## Current Sensing

Kelvin-connected sense resistors SRAH/SRAL (phase A) and SRBH/SRBL (phase B). Peak differential threshold voltage ≈ 325 mV (GLOBALSCALER = 0, CS = 31, sin_x = 248).

**Current formula:** I_RMS = (GLOBALSCALER/256) × (CS/32) × (VFS / R_SENSE) / √2

GLOBALSCALER (0x0B): 8-bit global scalar; 0 = full scale (256); valid range 32–255; values > 128 recommended. IRUN [12:8] and IHOLD [4:0] in IHOLD_IRUN (0x10): set IRUN = 16–31 for best microstep accuracy.

**Sense resistor vs. maximum RMS current (GLOBALSCALER = 0, CS = 31):**

| R_SENSE | I_RMS max |
|---------|-----------|
| 0.22 Ω | 1.1 A |
| 0.15 Ω | 1.6 A |
| 0.12 Ω | 2.0 A |
| 0.10 Ω | 2.3 A |
| 0.075 Ω | 3.1 A |
| 0.050 Ω | 4.7 A |
| 0.033 Ω | 7.1 A |
| 0.022 Ω | 10.6 A |

## Protection Features

- **Overtemperature prewarning (otpw):** 120 °C (typ); driver continues operating.
- **Overtemperature shutdown (ot):** 136 / 143 / 150 °C selectable via OTSELECT in DRV_CONF [17:16]; driver latches off until die cools below prewarning threshold.
- **Short to GND (s2ga, s2gb):** High-side detection; retries 3× before latching off.
- **Short to VS (s2vsa, s2vsb):** Low-side detection including sense resistor path.
- **Open load (ola, olb):** Informative only; no driver action; reliable only during slow SpreadCycle operation.
- **Charge pump undervoltage (uv_cp):** Flagged in GSTAT (0x01); clears on write-1.
- **DRV_ENN pin (active HIGH):** Disables all outputs within 3 clock cycles; hardware-independent of software.
- **Emergency stop via ENCA_DCIN:** GCONF stop_enable (bit 15) makes a high level on ENCA_DCIN trigger an abrupt motor stop and power-down state (IHOLD / IHOLDDELAY settings apply).
- **Power-on reset:** Chip resets defaults if VSA, 5VOUT, or VCC_IO falls below undervoltage threshold (VSA rising ≈ 4 V). VCC is not monitored; must not be lost during operation.

## Typical Applications

- **Industrial automation:** High-torque axes (3–10 A) with full motion profile and sensorless homing via StallGuard2.
- **Medical / laboratory equipment:** Precise positioning with encoder deviation monitoring (ENC_DEVIATION) and hardware emergency stop (DRV_ENN).
- **3D printing / CNC:** StealthChop2 for silent operation at low speed; SpreadCycle for high-speed moves; MicroPlyer for smooth motion from coarser step sources.
- **Robotics:** dcStep for load-adaptive velocity; CoolStep for energy efficiency at varying loads.

**Evaluation hardware:** TMC5160-BOB (breakout), TMC5160-EVAL (evaluation board), TMC5160-EVAL-KIT (full kit with Landungsbrücke + Eselsbrücke), TMC5160 SilentStepStick (20×15 mm Step/Dir module).

## Layout Considerations

- Connect all GND pins (GND, GNDP, GNDA, exposed die pad) to a common star-point GND plane directly beneath the IC.
- Place IC close to MOSFET bridge and sense resistors to minimize parasitic inductance and GND differences.
- Keep supply traces from filter capacitors to MOSFETs short and straight; use dedicated VS plane for high-current paths.
- 5VOUT: 2.2–4.7 µF ceramic capacitor at pin, GND return to GNDA only.
- VCC: 470 nF ceramic 1–2 mm from pin; couple to 5VOUT via 2.2–3.3 Ω resistor.
- VSA: 100 nF ceramic to GND plane.
- 12VOUT: 2.2–10 µF capacitor required.
- Bootstrap capacitors CA1/CB1/CA2/CB2: match to MOSFET specifications.
- TQFP package: apply PCB conformal coating for operation above 30 V to satisfy creep-distance requirements.

## Selection Notes

- **vs. [[TMC2209]]:** TMC5160A adds integrated ramp generator, higher voltage (60 V vs 29 V), and external MOSFET bridge for higher currents. Register set differs at the motion controller layer.
- **vs. TMC5130:** Pin and software compatible; TMC5160A supports higher motor currents via external MOSFETs. Same CHOPCONF/PWMCONF register layout.
- **vs. TMC5041/TMC5072:** Those are dual-motor variants with lower current; same ramp generator architecture.
- **vs. [[TMC2240]]:** TMC2240 integrates the bridge (lower BOM complexity); TMC5160A scales to higher currents and adds the full six-section ramp generator.
- **Higher voltage (> 60 V):** Use external gate driver ICs (e.g., MAX5062C) to boost gate outputs beyond the TMC5160A's 60 V absolute maximum.
