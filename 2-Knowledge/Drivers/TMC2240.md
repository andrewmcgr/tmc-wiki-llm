## Overview

The TMC2240 is a high-integration 2-phase bipolar stepper motor driver from Trinamic/Analog Devices. It includes a full H-bridge power stage, **Integrated Current Sense (ICS)** via MOSFET RDS(ON) (no external sense resistors required), an ABN incremental encoder interface, on-chip temperature sensing, and the full Trinamic motion suite: StealthChop2, SpreadCycle, MicroPlyer (256× interpolation), StallGuard2/SG4, and CoolStep. The device supports both SPI and single-wire UART with CRC.

## Interfaces

### SPI
- MODE 3 (CPOL=1, CPHA=1); max 10 MHz
- 40-bit datagrams: 8-bit address+R/W + 32-bit data; status byte returned on every access
- Status bits include: RESET, DRV_ERR, UV_CP, SG2 active, standstill

### UART (Single-Wire)
- Auto baud rate detection; CRC8-ATM polynomial
- Min baud at 20 MHz clock: 9000 baud; max: fCLK/16
- Up to 255 nodes on one wire (NODEADDR in NODECONF)
- SENDDELAY configurable for multi-node bus access

### STEP/DIR
- Standard step/direction inputs; MicroPlyer interpolates to 256 microsteps
- multistep_filt in GCONF filters noise on STEP input

## Non-Dissipative Integrated Current Sense (ICS)

The TMC2240 measures motor current by sampling the individual MOSFET RDS(ON) of the H-bridge — **no external sense resistors are required**, eliminating power dissipation in the sensing circuit.

Full-scale current is set by RREF (12–60 kΩ) and CURRENT_RANGE:

| CURRENT_RANGE | IFS (peak) | KIFS | RDS(ON) typ |
|---|---|---|---|
| 0x0 (default) | 1.0 A | 11.75 | 0.40 Ω |
| 0x1 | 2.0 A | 24 | 0.27 Ω |
| 0x2 / 0x3 | 3.0 A | 36 | 0.23 Ω |

**Formula:** IFS = KIFS / RREF

Current scaling: GLOBALSCALER (0x0B) — value 0 = full scale (256); valid range 32–255; values > 128 recommended for best accuracy. Per-motor tuning via IRUN and IHOLD (5-bit fields, 0–31) in IHOLD_IRUN (0x10).

## Motion Control Features

### MicroPlyer (256× Interpolation)
Hardware interpolation to 256 microsteps from any STEP resolution. Sine table is fully programmable: MSLUT[0..7] (0x60–0x67), MSLUTSEL (0x68), MSLUTSTART (0x69). OFFSET_SIN90 compensates phase shift between motor coils.

### StealthChop2
Voltage-mode PWM chopper for silent, low-vibration operation at low to medium speeds.
- Auto-tuning via `pwm_autoscale` + `pwm_autograd` (PWMCONF 0x70)
- 4 PWM frequency options (fCLK-based, ~20–50 kHz typical)
- PWM_OFS default = 30; PWM_SCALE (0x71) reports PWM_SCALE_SUM (0–1023)
- Automatic readback of tuned values via PWM_AUTO (0x72)
- FREEWHEEL options: normal / freewheeling / low-side short / high-side short
- Active below TPWMTHRS velocity threshold

### SpreadCycle
Cycle-by-cycle current chopper for precise current control at higher speeds.
- Enabled when chm = 0 in CHOPCONF (0x6C)
- Active between TCOOLTHRS and THIGH velocity thresholds

### Velocity-Based Mode Switching
Four programmable velocity thresholds (all in units of 1/TSTEP):

| Register | Address | Purpose |
|---|---|---|
| TPWMTHRS | 0x13 | Below this → StealthChop2 active |
| TCOOLTHRS | 0x14 | Below this → CoolStep/SG2 active |
| THIGH | 0x15 | Above this → fullstep mode |
| TSTEP | 0x12 | Measured actual step period (read-only) |

### StallGuard2 (SpreadCycle mode)
- SGT threshold: signed, −64 to +63 (COOLCONF 0x6D, sgt field)
- SG_RESULT: 0–1023, updated once per electrical fullstep; 0 = stall
- sfilt enables filtering; reported in DRV_STATUS (0x6F)

### StallGuard4 (StealthChop2 mode)
- SG4_THRS: 0–255 (register 0x74); sg4_filt_en for noise filtering
- SG4_RESULT: 0–510 (register 0x75)
- SG4_IND (0x76): individual quarter-bridge indicators SG4_IND_0..3
- sg_angle_offset in SG4_THRS for jerk-free mode switching between SG2/SG4

### CoolStep (Dynamic Current Reduction)
Reduces motor current dynamically based on load (StallGuard feedback). Up to 75% energy savings under light load.

Key COOLCONF fields:

| Field | Range | Description |
|---|---|---|
| SEMIN | 0–15 | 0 = disable; lower SG threshold (×32) |
| SEMAX | 0–15 | Upper SG threshold offset |
| SEUP | 0–3 | Current increment step (1/2/4/8) |
| SEDN | 0–3 | Decrement speed (32/8/2/1 readings) |
| SEIMIN | 0/1 | Minimum current: 1/2 or 1/4 of IRUN |

## Protection Features

| Feature | Details |
|---|---|
| Overcurrent (OCP) | Hardware limit; IMAX = 5.0 A; blanking 1.5 µs typ |
| Short-to-GND | Detected per bridge; diss2g disables if needed |
| Short-to-VS | Detected per bridge; diss2vs disables if needed |
| Thermal shutdown (TSD) | 165 °C; automatic restart after cooling (hysteresis 20 °C) |
| Over-temp prewarning (OTW) | Programmable threshold via OTW_OV_VTH (0x52) |
| Overvoltage (OV) | Open-drain OV pin; threshold set in OTW_OV_VTH |
| UVLO | VS < 3.9 V triggers undervoltage lockout |
| Charge pump UV | uv_cp flag in GSTAT (0x01) |
| DRV_ENN | Hardware enable pin; disables output stage when asserted |
| Emergency stop | stop_enable bit in GCONF (0x00) |

Diagnostic output pins DIAG0 and DIAG1 are individually configurable (error, stall, index, step, over-temp, etc.) via GCONF.

## Temperature Sensing

- **Chip temperature:** Internal ADC via ADC_TEMP (0x51); ±10 °C accuracy
- **Motor temperature:** Estimated from motor coil resistance change (referenced to GLOBALSCALER current)
- **Supply voltage monitoring:** ADC_VSUPPLY_AIN (0x50) — upper 16 bits = VS, lower 16 bits = AIN (0–1.25 V, VAIN = ADC_AIN × 305.2 µV)

---

## ABN Encoder Interface

- 3-channel incremental encoder (A, B, N)
- 32-bit position counter: X_ENC (0x39)
- ENC_CONST (0x3A): 16.16 signed fixed-point steps-per-encoder-pulse
- N-event latches X_ENC into ENC_LATCH (0x3C)
- ENCMODE (0x38): configures polarity (pol_A/B/N), edge sensitivity (pos_neg_edge), clear/latch behavior (clr_cont, clr_once, clr_enc_x), decimal/binary mode (enc_sel_decimal), ignore_AB

Use the TMC2240 when eliminating sense resistors is a priority, when StallGuard4 in StealthChop2 mode is required, or when on-chip encoder interface and temperature sensing reduce BOM complexity. Prefer [[TMC5160A]] for higher-voltage or higher-current applications with external MOSFETs.
